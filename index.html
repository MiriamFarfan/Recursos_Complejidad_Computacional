Módulo 1

¿Qué es un Algoritmo?

Antes de analizar la complejidad de algo, necesitamos saber exactamente qué estamos analizando. Vamos desde lo más básico.

👨‍🏫
Profe: Imagina que le das instrucciones a alguien para hacer un sándwich. Si las instrucciones son claras, siempre terminan, y siempre producen un sándwich... ¡eso es un algoritmo! Ahora formalicémoslo.
📖 Definición Formal
Un algoritmo es un procedimiento computacional que tiene:

1
Entrada bien definida
Recibe datos estructurados: números, arreglos, dataframes. No puede ser vago sobre qué acepta.
2
Pasos efectivos y claros
Cada paso puede ejecutarse mecánicamente, sin necesitar interpretación humana.
3
Salida bien definida
Produce un resultado que resuelve el problema planteado.
4
Terminación o convergencia
O bien termina en pasos finitos (ordenar un arreglo), o converge con criterio explícito (SGD: "para cuando cambio < ε").
5
Determinismo o control del azar
Si es estocástico (como random forest), la aleatoriedad debe controlarse con semilla.
⚠️ Error Común en Examen
Este código NO es un algoritmo:

mientras Verdadero:
imprimir "Hola"
¿Por qué? Porque no termina. Viola la propiedad de terminación.
💡 Ejemplo Real del PDF
¿Un sistema de recomendación activo 24/7 es un algoritmo?

❌ El sistema completo
Corre sin parar → NO termina → NO es un algoritmo en sentido estricto
✅ La rutina de recálculo
Recalcula cada hora, termina, produce resultado → SÍ es un algoritmo
🧠 ¿Por qué importa esta definición?

La definición formal nos permite hacer tres preguntas cruciales sobre cualquier algoritmo:

⚡ Eficiencia — ¿cuánto tarda?
📈 Escalabilidad — ¿funciona con más datos?
✅ Correctitud — ¿resuelve lo que dice?
🎯 Comprueba lo que aprendiste
¿Cuál de los siguientes NO es un algoritmo válido?
✅ ¡Correcto! El bucle infinito viola la propiedad de terminación. No importa que sus pasos sean claros — si nunca para, no es un algoritmo.
❌ Incorrecto. La respuesta correcta es B. Un bucle infinito viola la propiedad de terminación del algoritmo.
Módulo 2

Inducción Matemática

La herramienta más poderosa para probar que un algoritmo funciona para CUALQUIER tamaño de entrada, no solo para los que probaste.

👨‍🏫
Profe: Imagina una fila de dominós. Si el primero cae (caso base) y sabes que cuando cae cualquiera también cae el siguiente (paso inductivo)... ¿necesitas ver caer TODOS para saber que caerán todos? ¡No! Eso es inducción matemática.
🎯 Visualiza la Inducción

El primer dominó = caso base. Cada dominó que cae = paso inductivo.

📖 Estructura de una Prueba por Inducción
B
Caso Base
Verificar que P(n₀) es verdadero (usualmente n₀ = 0 ó 1). Es el "primer dominó".
H
Hipótesis Inductiva
Asumir que P(k) es verdadero para algún k ≥ n₀. No es trampa — es lo que se supone para demostrar el siguiente.
P
Paso Inductivo
Usando la hipótesis, demostrar que P(k+1) también es verdadero. Si esto se cumple, ¡ya probaste para todos los n!
✏️ Ejemplo 1: Suma de los primeros n enteros

Queremos probar que: 1 + 2 + 3 + ... + n = n(n+1)/2 para todo n ≥ 1

📐 Prueba completa
🔢 Verificar con números
🟢 CASO BASE — n = 1
Lado izquierdo: la suma solo tiene un término → 1
Lado derecho: aplicamos la fórmula → 1(1+1)/2 = 2/2 = 1
¿Son iguales? ✓ Sí. El caso base es verdadero.
🟡 HIPÓTESIS INDUCTIVA
Supón que para algún k ≥ 1, la fórmula ya es verdadera:
1 + 2 + ... + k = k(k+1)/2
→ Esto lo asumimos como "dado". Ahora usémoslo para el siguiente caso.
🟠 PASO INDUCTIVO — demostrar que vale para k+1
Queremos probar: 1 + 2 + ... + k + (k+1) = (k+1)(k+2)/2

Empezamos con el lado izquierdo y manipulamos:

1 + 2 + ... + k + (k+1)
   = k(k+1)/2 + (k+1)   [usamos la hipótesis]
   = (k+1)[k/2 + 1]   [factorizamos (k+1)]
   = (k+1)(k+2)/2   [simplificamos]

✅ Esto es exactamente lo que queríamos demostrar. ¡Prueba completa!

Mueve el slider para verificar la fórmula con valores concretos:

5
Suma real (1+2+...+n)
15
Fórmula n(n+1)/2
15
¿Son iguales?
✅
📋 Ejemplo 2 (del PDF): Probar que 3ⁿ > n³ para n ≥ 4
🟢 CASO BASE — n = 4
Calculamos ambos lados:
3⁴ = 3×3×3×3 = 81
4³ = 4×4×4 = 64
¿Es 81 > 64? ✓ Sí.
🟡 HIPÓTESIS INDUCTIVA
Asumimos que para algún k ≥ 4: 3ᵏ > k³
🟠 PASO INDUCTIVO — demostrar para k+1
Queremos probar: 3^(k+1) > (k+1)³

3^(k+1) = 3 · 3ᵏ > 3k³  [usamos hipótesis]

Ahora necesitamos ver si 3k³ > (k+1)³:
3k³ > k³ + 3k² + 3k + 1
2k³ - 3k² - 3k - 1 > 0

Para k=4: 2(64) - 3(16) - 3(4) - 1 = 128 - 48 - 12 - 1 = 67 > 0 ✓
Y la función crece para k ≥ 4, por lo tanto siempre se cumple. ✅ QED
🎯 Quiz — Inducción Matemática
En una prueba por inducción, la Hipótesis Inductiva sirve para...
✅ ¡Perfecto! La hipótesis inductiva es el "trampolín": la asumes verdadera para k y la usas para demostrar que vale para k+1. No es trampa, es lógica.
❌ La respuesta es C. La hipótesis inductiva asume P(k) para luego demostrar P(k+1).
Módulo 3

Definiciones Recursivas

Una función se define en términos de sí misma, con casos más pequeños. Es la base de Merge Sort, árboles de decisión y muchos algoritmos en ML.

👨‍🏫
Profe: Para entender la recursión, primero tienes que entender la recursión. 😄 En serio: una definición recursiva siempre tiene caso base (para cuando es trivial) y paso recursivo (que reduce el problema a uno más pequeño).
📖 Estructura de una Definición Recursiva
1
Caso Base
Una condición inicial que define el valor mínimo sin recurrir a sí misma. Sin él, ¡la recursión nunca termina!
2
Paso Recursivo
Define f(n) usando f(n-1) o instancias más pequeñas. Cada llamada se acerca al caso base.
🔢 Factorial: n! — Ejemplo Clásico
0! = 1     (caso base)
n! = n · (n-1)!     para n ≥ 1 (paso recursivo)

Elige un valor de n y ve cómo se "desenrolla" la recursión:

🌀 Fibonacci — Recursión con Dos Casos Base
F(0) = 0     F(1) = 1     (dos casos base)
F(n) = F(n-1) + F(n-2)    para n ≥ 2
👨‍🏫
Profe: Fibonacci necesita dos casos base porque F(n) depende de los dos anteriores. Si solo tuviéramos F(0)=0, intentaríamos calcular F(1) = F(0) + F(-1)... ¡y F(-1) no está definido!
📐 Prueba con inducción fuerte (del PDF)
Probar: F(n) < 2ⁿ para todo n ≥ 0

🟢 CASOS BASE
n=0: F(0) = 0 < 1 = 2⁰ ✓
n=1: F(1) = 1 < 2 = 2¹ ✓
🟡 HIPÓTESIS FUERTE
Supón F(k) < 2ᵏ para todo k < n (inducción fuerte: asumimos para todos los anteriores)
🟠 PASO INDUCTIVO
F(n) = F(n-1) + F(n-2)
< 2^(n-1) + 2^(n-2)  [hipótesis]
= 2^(n-2) · (2 + 1) = 3 · 2^(n-2)
< 4 · 2^(n-2) = 2ⁿ  ✅
🎯 Quiz — Recursividad
¿Cuál es el resultado de calcular la secuencia a₅ si a₁ = 1 y aₙ = 2aₙ₋₁ + 1 para n ≥ 2?
✅ ¡Correcto! a₂=3, a₃=7, a₄=15, a₅=31. En general aₙ = 2ⁿ - 1 (¡probado por inducción en el PDF!)
❌ Calculemos paso a paso: a₂=2(1)+1=3, a₃=2(3)+1=7, a₄=2(7)+1=15, a₅=2(15)+1=31. Respuesta D.
Módulo 4

Notación Asintótica

O, Ω, Θ, o, ω — el lenguaje para comparar qué tan rápido crecen los algoritmos cuando los datos crecen.

👨‍🏫
Profe: Imagina dos autos. No me importa cuál va más rápido en un semáforo. Me importa: si duplicamos la autopista, ¿cuál llega antes? La notación asintótica describe el comportamiento cuando n se vuelve muy grande, ignorando constantes y términos pequeños.
O(g)
Cota Superior
"f crece a lo mucho tan rápido como g"
∃ c,n₀ : f(n) ≤ c·g(n) para n ≥ n₀
Ω(g)
Cota Inferior
"f crece al menos tan rápido como g"
∃ c,n₀ : c·g(n) ≤ f(n) para n ≥ n₀
Θ(g)
Cota Ajustada
"f crece exactamente igual que g"
f = O(g) Y f = Ω(g)
o(g)
Cota Superior Estricta
"f crece estrictamente más lento que g"
lím f(n)/g(n) = 0
ω(g)
Cota Inferior Estricta
"f crece estrictamente más rápido que g"
lím f(n)/g(n) = ∞
🎯 Truco con Límites (del PDF)

Para clasificar f(n) respecto a g(n), calcula lím[n→∞] f(n)/g(n):

Resultado del límiteRelaciónSignifica
= 0f = o(g) y también f = O(g)f crece más lento
= c > 0f = Θ(g)f y g crecen igual
= ∞f = ω(g) y también f = Ω(g)f crece más rápido
📊 Jerarquía de Complejidades
20
✏️ Ejemplo resuelto (del PDF): f(n) = 4n³ + 2n² + n + 5
👨‍🏫
Profe: Cuando tenemos una suma de términos, el término de mayor grado domina cuando n crece. Los demás se vuelven insignificantes.
Demostrar que f(n) = O(n³)
Para n ≥ 1, cada término se puede acotar por n³:

4n³ ≤ 4n³
2n² ≤ 2n³  [porque n² ≤ n³ para n ≥ 1]
n ≤ n³
5 ≤ 5n³

Sumando: 4n³ + 2n² + n + 5 ≤ (4+2+1+5)n³ = 12n³

Por lo tanto: f(n) = O(n³) con c = 12, n₀ = 1 ✅
🎯 Quiz — Notación Asintótica
Sea f(n) = n log n. ¿Cuál es la relación correcta con g(n) = n²?
✅ ¡Excelente! lím[n·log(n)/n²] = lím[log(n)/n] = 0 (L'Hôpital: 1/n → 0). Como el límite es 0, f es small-o y también Big-O de g.
❌ La respuesta es A. El límite de (n·log n)/n² = log(n)/n → 0, lo que significa f(n) = o(n²).
Módulo 5

Análisis de Algoritmos

Aprende a leer pseudocódigo y derivar su complejidad Big-O línea por línea, como lo hace el profesor en pizarrón.

👨‍🏫
Profe: La regla de oro es: cuenta cuántas veces se ejecuta cada línea y suma todo. Luego te quedas con el término que crece más rápido.
📏 Reglas de Conteo
EstructuraComplejidadRazón
Asignación simpleO(1)1 operación siempre
if/elseO(max(rama1, rama2))Se ejecuta una sola rama
for i = 1 to nO(n)n iteraciones
Bucles anidados (n×n)O(n²)n×n operaciones
while i ≤ n; i=i*2O(log n)i: 1,2,4,8...→ log₂(n) pasos
Dos bucles seguidosO(max(T₁, T₂))Término dominante
🎬 Pseudocódigo Animado — Correlaciones (del PDF)

Presiona ▶ para analizar línea por línea:

Listo
1n ← len(dataset)
2m ← n_features(dataset)
3for i ← 1 to m do
4for j ← i+1 to m do
5corr ← 0
6for k ← 1 to n do
7corr ← corr + dataset[k,i] × dataset[k,j]
8correlation[i,j] ← corr/n
💡 Análisis
Resultado: O(m²·n)
Con d variables y n filas: O(d²n) — cuadrático en variables, lineal en datos
🔍 Búsqueda Binaria — O(log n)
👨‍🏫
Profe: Cada iteración divide el espacio de búsqueda exactamente a la mitad. Si empezamos con n elementos, tras 1 paso tenemos n/2, tras 2 tenemos n/4... ¿cuántos pasos hasta llegar a 1? Exactamente log₂(n).
T(n) = T(n/2) + c   →   T(n) = Θ(log n)
🔀 Merge Sort — O(n log n)
👨‍🏫
Profe: Merge Sort es divide y vencerás. Divide el arreglo en mitades (log n niveles), y en cada nivel hace O(n) trabajo de fusión. Total: n × log n. El árbol de recursión lo hace muy visual.
T(n) = 2T(n/2) + O(n)  →  Teorema Maestro caso 2  →  Θ(n log n)
🎯 Quiz — Análisis Big-O
¿Cuál es la complejidad de este pseudocódigo?

i ← 1
while i < n do
  imprimir i
  i ← i * 2
end while
✅ ¡Correcto! i toma los valores 1, 2, 4, 8, 16... (potencias de 2). Termina cuando i ≥ n, es decir tras log₂(n) iteraciones → O(log n).
❌ La respuesta es B: O(log n). Como i se duplica en cada paso (i=1,2,4,8...), solo hacemos log₂(n) iteraciones.
Módulo 6

Grafos y sus Representaciones

Estructuras fundamentales para modelar redes sociales, recomendaciones, fraude financiero y más.

👨‍🏫
Profe: Un grafo es solo puntos conectados por líneas. Los puntos son vértices y las líneas son aristas. Formalmente: G = (V, E). Simple. Pero poderoso.
Grafo No Dirigido
Las aristas no tienen dirección. Si A conecta con B, entonces B conecta con A.

Ejemplo: amistades en Facebook
E ⊆ {{u,v} | u,v ∈ V}
Grafo Dirigido
Las aristas tienen dirección (arcos). A→B no implica B→A.

Ejemplo: seguidores en Twitter
E ⊆ V × V (pares ordenados)

🎯 Quiz — Grafos
En el ejemplo del PDF: usuario A ve X e Y, usuario B ve Y y Z, usuario C ve X. ¿Qué tipo de grafo es más apropiado para modelar este sistema de recomendación?
✅ ¡Correcto! Los sistemas de recomendación usan grafos bipartitos: un conjunto de nodos son usuarios {A,B,C} y el otro son películas {X,Y,Z}. Las aristas solo van de usuarios a películas.
❌ La respuesta es C: grafo bipartito. Los dos conjuntos disjuntos son usuarios y películas. Las aristas representan "usuario vio película".
Módulo 7

Ordenamiento y Recurrencias

Merge Sort, Búsqueda Binaria y el Teorema Maestro: los tres pilares del análisis de algoritmos recursivos.

👑 Teorema Maestro
👨‍🏫
Profe: Cuando un algoritmo recursivo hace a llamadas de tamaño n/b y el trabajo extra es f(n), el Teorema Maestro te da la complejidad en un solo paso.
T(n) = aT(n/b) + f(n)
a ≥ 1 subproblemas, de tamaño n/b, costo extra f(n)
CASO 1: f(n) = O(n^c) con c < log_b(a)
Los subproblemas dominan → T(n) = Θ(n^(log_b a))
CASO 2: f(n) = Θ(n^(log_b a))
Ambos empatan → T(n) = Θ(n^(log_b a) · log n)
CASO 3: f(n) = Ω(n^c) con c > log_b(a)
El trabajo extra domina → T(n) = Θ(f(n))
✏️ Merge Sort — Aplicar el Teorema
T(n) = 2T(n/2) + O(n)

a = 2, b = 2, f(n) = n
c_crítico = log₂(2) = 1
f(n) = n = Θ(n¹) → Caso 2

Resultado: T(n) = Θ(n log n) ✅
✏️ Búsqueda Binaria — Aplicar el Teorema
T(n) = T(n/2) + c

a = 1, b = 2, f(n) = c (constante)
c_crítico = log₂(1) = 0
f(n) = O(n⁰) → Caso 2

Resultado: T(n) = Θ(log n) ✅
🎯 Quiz — Teorema Maestro
Aplica el Teorema Maestro a: T(n) = 4T(n/2) + n
✅ ¡Correcto! a=4, b=2, c_crítico = log₂(4) = 2. f(n) = n = O(n¹) y 1 < 2 → Caso 1 → T(n) = Θ(n²).
❌ Respuesta: B — Θ(n²). a=4, b=2, c_crítico = log₂(4) = 2. f(n)=n=O(n¹), y 1 < 2 → Caso 1 → T(n) = Θ(n^(log₂4)) = Θ(n²).

Repaso Completo

Preguntas integradoras de todos los temas de la Unidad 1. ¡Prepárate para el examen!

Pregunta 1 de 4 — Inducción
¿Qué debes probar en el paso inductivo para demostrar que P(n) vale para todo n ≥ 1?
✅ El paso inductivo demuestra que si P(k) es verdad → P(k+1) también. Esa implicación, junto al caso base, garantiza que vale para todos los naturales.
❌ Respuesta D. El paso inductivo prueba P(k) ⟹ P(k+1), no verifica casos individuales.
Pregunta 2 de 4 — Notación Asintótica
¿Cuál afirmación es FALSA?
✅ n³ NO es O(n²). Para serlo, necesitaría existir c tal que n³ ≤ c·n², es decir n ≤ c para todo n grande. Imposible.
❌ Respuesta B. n³ = O(n²) es falso: n³/n² = n → ∞, no está acotado.
Pregunta 3 de 4 — Análisis
¿Cuánto cuesta (Big-O) calcular la media acumulativa de n elementos usando la fórmula recursiva μₙ = ((n-1)μₙ₋₁ + xₙ) / n?
✅ Cada actualización de la media es O(1) y hacemos n de ellas → O(n) total. Mucho mejor que recalcular la media desde cero en cada paso (O(n²)).
❌ Respuesta A — O(n). Cada paso hace trabajo constante O(1), y hay n pasos.
Pregunta 4 de 4 — Grafos
En un grafo no dirigido con n vértices, ¿cuánto espacio ocupa la matriz de adyacencia?
✅ La matriz es de tamaño n×n = n² celdas. Por eso para grafos con pocos aristas (dispersos) es mejor la lista de adyacencia O(V+E).
❌ Respuesta C — O(n²). La matriz tiene n filas × n columnas.
